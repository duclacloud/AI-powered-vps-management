The command `systemctl status nginx` is used to check the **status of the Nginx service** managed by **systemd**. Here's a breakdown of what it does:

1. **Service Status Check**:
   It displays whether the Nginx service is **active (running)**, **inactive (stopped)**, or in a **failed state**.
   Example output:
   ```
   ● nginx.service - High-performance web server and proxy server
        Loaded: loaded (/lib/systemd/system/nginx.service; enabled; vendor preset: enabled)
        Active: active (running) since Fri 2023-10-06 14:30:00 UTC; 5min ago
   ```

2. **Process Information**:
   Shows the **main process ID (PID)** of Nginx and its child processes (workers).
   Example:
   ```
   Main PID: 1234 (nginx)
   ```

3. **Recent Activity**:
   Displays **start/stop times** and reasons for failures (if applicable).
   Example:
   ```
   Lines: 25 (Partial Output)
   ```

4. **Logs and Errors**:
   Provides details on **recent log entries** or errors (if the service is active and logging to systemd).
   Example:
   ```
   Oct 06 14:30:00 server nginx[1234]: 2023/10/06 14:30:00 [error] ...
   ```

5. **Reload Configuration**:
   If Nginx is reloaded (`nginx -s reload`), the status command can confirm the new configuration is active.

### Use Cases:
- Verify if Nginx is running properly.
- Troubleshoot service failures (e.g., crashes, configuration errors).
- Check resource usage (indirectly via process details).
- Confirm post-restart status.

### Notes:
- Requires `sudo` if run by a non-root user (e.g., `sudo systemctl status nginx`).
- Output may vary based on system configuration and Nginx setup.
